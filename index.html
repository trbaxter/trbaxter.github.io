Projects

York Forums

My first full stack web application created in the style of an online forum resembling Reddit.
This was submitted for the final assignment of the York Solutions apprenticeship program.

Note:  Project was for educational purposes only. No official identification with York Solutions is intended or should be inferred.

  Project Overview

Allows users to register, login, and logout, as well as perform CRUD (Create/Read/Update/Delete) operations on topic threads to foster online discussion.

  Front-end Tech Stack & Purposes

CSS: General-purpose appearance modification.

HTML: Used to host the webpage design and contain the JavaScript.

JavaScript: Used in conjunction with React to drive application logic.
(e.g. State-Reducer model)

Redux: Controller for managing application state and state changes.

Bootstrap: Used for quick CSS templating of input forms and buttons.

Back-End Technologies & Purposes

Java: Used to drive the logic for REST endpoints and database functionality.

Spring Boot: Provided REST boilerplate configuration.

Gradle: Used to automate the build and compile actions of the project.

PostgreSQL: Stored user profiles and forum thread information.

Docker: Used for management and containerization of PostgreSQL data.

Postman: Used for endpoint setup verification.

Assignment Statement

Develop a functional full stack web application using the specified tech stack to demonstrate proficiency in building robust, secure, and scalable web solutions.

Required features:

  1. User Authentication
    • Design a main sign-in page allowing a user to sign up by creating a username and password.
    • Implement user authentication functionality by enabling users to log in with their credentials.
    • Prevent further access into the application without properly identified credentials.
  2. Data Persistence
    • Store user information securely in the PostgreSQL database for future retrieval.
    • Assign a unique UUID to each user account for identification purposes.
  3. User Interaction
    • Enable logged-in users to perform CRUD operations on the application state.
    • Ensure that changes made by users persist even after logout and are visible to other users upon login.
  4. User Interface
    • Define the user interface using JavaScript, ensuring that HTML is generated dynamically through JavaScript functions.
  5. Dependency Restriction
    • Strictly adhere to the specified tech stack; no additional dependencies are allowed.
